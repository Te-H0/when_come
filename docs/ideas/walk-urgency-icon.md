# 위치 기반 "뛰어야 돼?" 사람 아이콘 애니메이션

작성일: 2026-05-10
상태: 검토 필요

## 컨셉

홈 화면 "지금 타야할 교통수단" 카드에서, 도착시간 숫자만 보여주는 대신 사람 아이콘이 뛰는 속도를 다르게 해서 "지금 뛰어야 하는지 / 여유 있는지"를 직관적으로 보여줌.

대상: 경로(routes)의 현재 스텝 카드. 즐겨찾기는 대상 아님 (출발지 컨텍스트가 없으므로).

## 핵심 로직

```
여유 = (도착까지_분 - 정류장까지_도보_분) / 도보_분

여유 ≥ 1.0  → 🚶  천천히 (여유 충분)
여유 ≥ 0.3  → 🚶‍♂️  보통
여유 ≥ 0    → 🏃  빨리
여유 < 0    → 🏃‍♂️💨  뛰어 (이미 늦음, 다음 차 추천)
```

## 구현 검토 포인트

### 거리 계산
- **MVP:** 직선거리(Haversine) × 1.3 보정, 도보속도 4km/h 고정
- **개선 옵션:** 네이버/카카오 도보경로 API (정확도 ↑, 카드마다 호출 비용 ↑)
- 정확도 불만 신호 누적되면 그때 도보 API 도입

### 위치 권한
- 첫 진입 시 1회 요청 (Geolocation API)
- 거부 시 fallback: 기존 카드 그대로, 아이콘만 빠짐
- 권한 거부 후 재요청은 설정 안내만

### 위치 갱신 주기
- 화면 진입 1회 + 30초 polling + 새로고침 버튼 시
- `watchPosition`은 배터리 소모 커서 비추

### 어디에 띄울지
- 현재 스텝(첫 카드)만 — 다음 스텝은 환승 후 거리라 의미 없음
- 단일 stepGroup에 정류장 2개면? 둘 다 띄움 (각 카드별 거리 계산)

### 안전장치
- **정류장 50m 이내** (GPS 오차 범위) → 아이콘 숨기고 "도착함" 메시지로 대체
- 그렇지 않으면 GPS 오차로 "뛰어!" ↔ "천천히"가 깜빡일 수 있음
- 정류장 좌표 없는 경우 (legacy data) → 아이콘 미표시

## V2 확장 아이디어

- "지금 뛰면 643번, 천천히 가도 651번" 같은 노선 추천
- 사용자별 도보속도 학습 (실제 도착 시간 vs 예측 비교)
- 다음 스텝 환승까지 합산해 "환승 여유" 표시

## 차별화 가치

- 네이버/카카오는 도착시간 숫자만 보여줌
- 사용자가 머릿속에서 매번 하는 계산 ("6분 후 도착, 정류장까지 4분 거리, 여유 있네")을 시각화로 대신
- 출퇴근 앱 컨텍스트에 매우 적합 (하루 2번 반복되는 결정)
- "뛰어!" 모먼트는 공유성/입소문 포인트
